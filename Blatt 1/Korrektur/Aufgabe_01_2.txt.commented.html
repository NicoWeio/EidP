    Schema zur Berechnung: 
    - Die Zahl durch 2 dividieren
    - Den Rest der Division notieren
    - Falls das Ergebnis nicht 0 ist, Schritt 1 und 2 wiederholen
    - Das Ergebnis "rückwärts" ablesen

    15754 / 2 = 7877 Rest 0
    7877 / 2 = 3938 Rest 1
    3938 / 2 = 1969 Rest 0
    1969 / 2 = 984 Rest 1
    984 / 2 = 492 Rest 0
    492 / 2 = 246 Rest 0
    246 / 2 = 123 Rest 0
    123 / 2 = 61 Rest 1
    61 / 2 = 30 Rest 1
    30 / 2 = 15 Rest 0
    15 / 2 = 7 Rest 1
    7 / 2 = 3 Rest 1
    3 / 2 = 1 Rest 1
    1 / 2 = 0 Rest 1

    15754_10 = 11110110001010_2

b)
n=1 ist in Binärdarstellung zuerst …01
Für jede Multiplikation mit 2 (Schritt 4) verschiebt sich diese 1 eine Stelle nach links.
Da die Schleife 31-mal ausgeführt wird, befindet sich, solange kein Überlauf geschieht, an (31+1)-ter Stelle von rechts die 1.

1. 
    Es sind genügend Stellen vorhanden, um die Zahl korrekt abzubilden. Das Ergebnis ist daher:
    2^31 = 2147483648


2. 
    Die erste (höchstwertige) Stelle hat bei signed int's ein negatives Vorzeichen. Diese Stelle würde aber mit positivem Vorzeichen gebraucht, um die Zahl korrekt darzustellen. Dementsprechend ist das Resultat:
    -2^31 = -2147483648


c)
Wenn die zwei niedrigstwertigen Bits 0 sind, kann nur jede vierte Zahl dargestellt werden. Denn zählt man im Dualsystem von 0 an hoch, so durchlaufen die zwei niedrigstwertigen Bits immer wieder "00", "01", "10", und "11". Es lassen sich also nur alle x ∈ 4n, n ∈ ℕ darstellen.
Bei einer maximalen Anzahl von Bits ist die größtmögliche Zahl mit der geforderten Einschränkung um 3 geringer.

Allgemein lassen sich, wenn die k niedrigstwertigen Bits 0 sind, nur alle x ∈ (2^k)*n, n ∈ ℕ darstellen.
