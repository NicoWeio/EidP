a)
Eine Klasse ist ähnlich wie ein struct ein vom Programmierer definierter
Datentyp. Die Klasse beinhaltet also (i.d.R.) Attribute mit eigenen Datentypen.
Zusätzlich beinhalten Klassen Methoden, d.h. auf eine Instanz anwendbare Operationen.
Als ein Objekt wird eine einzelne Instanz einer Klasse mit konkret definierten
Werten bezeichnet.

b)
Wenn ein struct verwendet wird, sind alle Komponenten automatisch ???, während sie bei
der Verwendung von class private sind, bis sie vom Programmierer als public gekennzeichnet werden

c)
Konstruktoren werden beim Erstellen eines Objektes aufgerufen,
und ermöglichen erweiterte Funktionalität bei der Initialisierung eines Objektes.
Beispielsweise könnte sich ein neu erstelltes Objekt bei einer verwaltenden Instanz anmelden.
Der Destruktor wird aufgerufen, sobald der Gültigkeitsbereich des Objekts verlassen wird.
Zusammen können Konstruktor und Destruktor genutzt werden, um innerhalb von Objekten
mit dynamischem Speicher zu arbeiten.

d)
Nach dem Prinzip des "information hiding" wird die Definition der Klasse von ihrer
Implementierung getrennt, indem diese in einer anderen Datei stattfindet.


e)
Es können mehrere Methoden mit gleichem Namen definiert werden. Das nennt man
Überladen der Methode. Die gleichnamigen Methoden können vom Compiler
durch die unterschiedlichen Argumente unterschieden werden.

f)
Der Zweck dieser Teilung besteht darin, dass nur intern benötigte
Attribute und Methoden nicht von außerhalb zugänglich sind.
Sie sind damit geschützt vor (meist) versehentlicher fehlerhafter Manipulation von anderswo.
Es wird also abgesichert, dass auf das Objekt nur über die dafür vorgesehenen
"Schnittstellen"-Methoden zugegriffen wird.

g)
Von einer Delegierung spricht man, wenn ein Konstruktor einen anderen Konstruktor aufruft,
und dadurch die Arbeit der Initialisierung einer Instanz in gewisser Weise delegiert.
Dadurch kann z.B. überflüssiger Code vermieden werden.

h)
Bei Verwendung von Templates wird vom Compiler automatisch eine konkrete Klasse erzeugt. Daher
werden in diesem Fall Definition und Implementierung nicht nach dem Prinzip des "information hiding"
getrennt, sondern finden in der Header-Datei statt.
