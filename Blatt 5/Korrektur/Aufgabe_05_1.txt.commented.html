a)
Eine Funktionsschablone ist eine Funktion, die für verschiedene Datentypen aufgerufen werden kann.
Der Compiler generiert die benötigten spezifischen Funktionen daraus dann automatisch.
Hier ein Beispiel:

"traditionell":

	int quadrat(int x)
	{
		int y = x*x;
		return y;
	}

Diese Funktion funktioniert nur für ganze Zahlen;
für z.B Kommazahlen müsste eine separate Funktion definiert werden.

Mithilfe der Funktionsschablone:

	template <typename T>
	T quadrat(T x)
	{
		T y = x*x
		return y;
	}

Diese Funktion kann direkt mit allen (sinnvollen) Datentypen aufgerufen werden.


b)
Soll die Funktion ein mehrdimensionales Array entgegennehmen,
so müssen die Größen aller Dimensionen außer der ersten bekannt sein,
damit die Adressen der einzelnen Elemente des Arrays korrekt berechnet werden können.
Dabei macht es keinen Unterschied, ob das Array statisch oder dynamisch ist. Das gilt nur für statische Arrays 
Wird das Array allerdings nur als Pointer (auf das erste Element) entgegengenommen,
so müssen keine Größen der Dimensionen angegeben werden: die Berechnung der Speicheradressen Das funktioniert nur bei dynamischen Arrays
übernimmt dann der Programmierer.
Wie gewohnt sollte auch die Länge des Arrays, beziehungsweise die Größe
der einzelnen Dimensionen mit übergeben werden.
(-0.1 P.)

d)
inline-Funktionen werden vom Compiler (meist) in den beinhaltenden Block eingebunden,
fast als wären sie dort hineingeschrieben worden. Das beschleunigt das Programm,
da im Gegensatz zu "normalen" Funktionsaufrufen keine extra
Operationen (Argumente kopieren, …) erfolgen müssen,
geht jedoch zu Lasten des Bedarfs an Speicherplatz.
inline-Funktionen sind daher sinnvoll für kleine, häufig gebrauchte Operationen,
bei denen die Speicheroperationen, die für einen Funktionsaufruf benötigt werden,
sonst ins Gewicht fallen würden.
