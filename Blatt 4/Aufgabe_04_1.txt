a)
Funktionsdeklaration:
Rückgabedatentyp Funktionsname (Datentyp des Parameters Name des Parameters);

Funktionsdefinition:
Rückgabedatentyp Funktionsname (Datentyp des Parameters Name des Parameters) {
  Anweisungen…
  return Rückgabewert;
}

Beispiel:
int foo(int arg) {
  return arg*2; //Rückgabewert ist ein int
}

b)
Da die Länge des übergebenen Arrays für die Funktion nicht direkt erkennbar ist,
sollte die Länge (Bereichsgrenze) als Argument mit übergeben werden,
um Bereichsüberschreitungen zu verhindern. Außerdem ist bei der Übergabe zu beachten,
dass Arrays nicht als Kopie, sondern als Referenz übergeben werden.

c)
Wenn eine Variable als Wert übergeben wird, wird dieser als Kopie in einer lokalen Variable
übergeben, daher hat eine Veränderung der übergebenen Variable in der Funktion keinen Einfluss
auf den Wert der Variablen außerhalb der Funktion.
Wenn die Variable als Zeiger übergeben wird, wird zwar die Adresse der Variable als Kopie übergeben,
eine Änderung des referenzierten Wertes gilt jedoch auch außerhalb der Funktion.
Statt durch Zeiger kann der Wert der Variablen auch als Referenz übergeben werden.
In diesem Fall wird der Name des Parameters wie ein zweiter Name der übergebenen Variable behandelt.

d)
Das Argument wird als Kopie in einer lokalen Variable im Gültigkeitsbereich der Funktion gespeichert,
und kann daher nicht verändert werden. Es muss darauf geachtet werden,
dass die Datentypen von Argument und Parameter übereinstimmen.

c)
"return 66;" bedeutet, dass die Funktion den Wert 66 als unsigned char zurückgibt.
Die 66 ist die numerische Repräsentation des Buchstaben "B" gemäß der ASCII-Zeichentabelle.
Wenngleich innerhalb des Programms chars wie reguläre Zahlen auftreten, werden sie daher als Zeichen ausgegeben.
Diese Rückgabe kann entweder in einer anderen Variable des gleichen Datentyps abgespeichert, oder ohne
Zwischenspeicherung als konstanter Wert weiterverwendet werden.
